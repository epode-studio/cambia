# Introduction

**Cambia** is a [DESIGN.md](https://github.com/google-labs-code/design.md) extension and a live,
on-device engine. It adds two things on top of your design system:

1. **Roles** — what each component *does* (`primary-action`, `tabular-list`, `form-field`, …) and,
   per role, which traits are **conserved** (the grammar, never moves) vs **adaptive** (the runtime
   may personalize).
2. **A runtime** — that makes the adaptive traits actually adapt: every component starts
   *born-adapted* to the app's archetype, then personalizes to each user **on their device**.

It rides inside the DESIGN.md you already have, as one top-level `cambia:` key — so the file still
passes `npx @google/design.md lint`, and is read more deeply by `npx @epode/cambia check`.

## The layer stack

| Layer | Owned by | Lives in | Buys you |
| --- | --- | --- | --- |
| **L0 — visual identity** | DESIGN.md | front-matter tokens + prose | on-brand UI |
| **L0.5 — semantic roles** | Cambia | `cambia.roles` | the right component for the job; conserved/adaptive declared |
| **L1 — born-adapted + personalization** | `@epode/cambia-runtime` | priors + on-device state | components tuned to the app, then to each user |

## Install

```bash
npx @epode/cambia init     # add a cambia: block to your DESIGN.md
npx @epode/cambia check    # validate it
```

Then read [Getting started](/docs/getting-started).

## Packages

- [`cambia`](/docs/cli) — the CLI (`init` · `check` · `tailwind` · `skill`)
- [`@epode/cambia-core`](/docs/core) — parse / scaffold / validate (framework-zero)
- [`@epode/cambia-runtime`](/docs/runtime) — the live, on-device engine
- [`@epode/cambia-react`](/docs/react) — the `useCambia()` hook
